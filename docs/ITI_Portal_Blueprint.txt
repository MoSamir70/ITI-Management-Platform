1. System Overview & Roles

The ITI Branch Portal is a unified web platform managing the full lifecycle of ITI training tracks: from branch and intake setup, through academic delivery, to student KPI tracking. It is built on ASP.NET Core (backend) and Angular 17+ (frontend) with JWT-based authentication and Role-Based Access Control (RBAC).

1.1  The Five Roles
Role	Description
Training Manager	Super-admin. Creates branches, intakes, tracks, and assigns supervisors to tracks. Has full read/write access to every module.
Supervisor	Owns one track (per intake). Manages courses, enters grades, publishes grades and exam dates, approves absence requests and KPIs.
Teacher Assistant	Assigned to one or more groups within a track. Enters lab grades and records attendance for the courses/groups assigned.
Student Affairs	Creates and manages student accounts. Views student personal data, course tables, and absence records. Cannot view grades. Can view exam schedules (including corrective exams) for monitoring purposes.
Student	Views own course table, published grades, attendance record, exam notifications. Submits absence requests and KPIs.

1.2  Key Design Principles
•Training Manager creates all tracks and assigns exactly one supervisor per track-intake.
•Student Affairs creates all student accounts with personal enrolment data.
•Supervisors can view the personal data of students enrolled in their track.
•Student Affairs can view exam schedules (regular and corrective) for monitoring — they cannot modify them.
•Teacher Assistants can view student grades for their assigned courses only (read-only).
•No role can access data outside its defined scope; all API routes are protected by role claims.

3. Organisational Structure

The portal hierarchy has four levels: Branch → Intake → Track → Group. This structure supports multiple intakes of the same track running concurrently within the same branch.

3.1  Hierarchy
Branch  ›  Intake  ›  Track  ›  Group  ›  Students / Courses / TAs

Example:
Branch	Track	Intake	Groups
Alexandria	Full Stack .NET	Intake 45	Group A, Group B, Group C
Alexandria	Full Stack .NET	Intake 46	Group A, Group B
Alexandria	AI / Data Science	Intake 45	Group A

3.2  Rules
•A Branch has many Intakes.
•An Intake contains one or more Tracks (e.g. Full Stack .NET Intake 45, AI Intake 45).
•A Track in an Intake is assigned exactly one Supervisor by the Training Manager.
•A Track in an Intake can have multiple Groups (e.g. Group A, Group B, Group C).
•Each Group has its own set of Teacher Assistants (assigned by the Supervisor or Training Manager).
•Each Group has its own student roster; a student belongs to exactly one Group.
•Courses are defined at the Track level and are shared across all groups in that track.
•Grades and attendance are recorded at the student level within a course and group.

4. Authentication & User Management

4.1  Auth Flow
1.User navigates to /login.
2.Submits email + password → POST /api/auth/login.
3.Server validates credentials via ASP.NET Identity.
4.Server issues JWT access token (15 min) + refresh token (7 days).
5.Angular stores access token in memory; refresh token in httpOnly cookie.
6.HTTP interceptor attaches Bearer token to every outgoing request.
7.Route guard reads role claim from token; redirects if unauthorised.
8.On token expiry, interceptor silently calls /api/auth/refresh.

4.2  User Entity
Entity: Users (extends ASP.NET Identity)
Column	Type	Notes
Id	UNIQUEIDENTIFIER	ASP.NET Identity PK
FullName	NVARCHAR(200)

Notes:
•Student Affairs creates student accounts (personal data: name, national ID, phone, email, gender, DOB, group assignment).
•Training Manager creates staff accounts (supervisors, TAs, student affairs officers).
•Passwords are hashed via ASP.NET Identity (PBKDF2). Plain-text passwords are never stored.

6.3  Grade Flow
15.TA enters lab grades for their group → saved as draft.
16.Supervisor enters exam/other grades per course.
17.Supervisor reviews combined grades; may edit TA lab grades if needed.
18.Supervisor toggles 'Publish' → IsPublished = true, PublishedAt = now.
19.Students see grades in their portal immediately after publication.
•Training Manager can perform any step above across all tracks.
•TA can view (read-only) grades for their course — they cannot publish.

9. KPI Module

Two KPI types exist: Certificate and Freelance. Each type can be enabled at the track level, at the individual course level, or both. The supervisor configures which KPI types are applicable.

9.1  Certificate KPI
•Supervisor enables certificate KPI on a track and/or specific courses.
•Student submits a certificate with the following data:
◦Certificate Name (title of the certificate)
◦Issuing Body (e.g. Coursera, AWS, Microsoft)
◦Issue Date
◦Expiry Date (optional)
◦Certificate File (PDF or image upload)
◦Which track/course requirement it fulfils

9.2  Freelance KPI
•Supervisor enables freelance KPI on a track and/or specific courses.
•Student submits a freelance project with the following data:
◦Project Title
◦Platform Used (e.g. Upwork, Fiverr, Direct client)
◦Client Contact (name + email or phone)
◦Project Description (what was delivered)
◦Payment Screenshot (image upload — proof of payment)
◦Amount Earned (optional, in EGP or USD)

10. Student Portal

Students access a role-specific dashboard after login. The following features are available:

Feature	Details
Course Table	View full course list for own track/intake/group. Shows lecture, lab, and self-study hours, grading mode, and instructor name.
My Grades	Visible only after supervisor publishes. Shows lab grade, exam grade, and total per course.
Attendance	Total absences per course. Online/offline split. Percentage vs. allowed threshold.
Absence Request	Submit date range + reason. Track approval status (Pending / Approved / Rejected).
Notifications	Exam dates, grade releases, KPI decisions, corrective exam alerts. Inbox-style list with read/unread status.
Certificate KPI	Submit certificate with file upload. View approval status and reviewer note.
Freelance KPI	Submit project with payment screenshot. View approval status.
My Profile	View own personal data: name, national ID, phone, email, group, intake, track.



11. Student Affairs Module

Student Affairs staff are responsible for student account creation and pastoral monitoring. They have no access to academic grades.

11.1  Capabilities
Capability	Details
Create Student Accounts	Full personal data: name, national ID, phone, email, gender, date of birth, photo, group assignment, intake.
View Student Profile	Read-only access to all personal data fields for any student.
View Course Table	See any student's track course list with hours breakdown. Cannot see grades.
View Absence Records	Online and offline absences per student per course. Compare to threshold.
Record Offline Absence	Can record in-person absences for any student.
View Absence Requests	See all submitted requests and supervisor decisions. Cannot approve.
View KPI Status	See submitted KPIs and whether supervisor approved or rejected. No action.
View Exam Schedules	Can see regular and corrective exam dates and locations for monitoring purposes. Read-only.
Student Search & Filter	Search by name, national ID, group, track. Filter by absence risk. Export reports.

RESTRICTION: Student Affairs CANNOT view student grades (published or draft), exam scores, or grade breakdowns.



12. Teacher Assistant Module

Teacher Assistants are assigned to one or more Groups within a track by the Supervisor or Training Manager. Multiple TAs can be assigned to the same group. Their scope is always limited to their assigned group(s).

12.1  Capabilities
Capability	Details
Enter Lab Grades	Enter and update lab grades for students in their assigned group for applicable courses.
View All Grades	Read-only view of all grades (lab + exam + total) for students in their assigned courses. Cannot publish.
Record Attendance	Record online and offline absence for students in their assigned group.
View Course Info	Read-only: course name, hours, instructor, grading mode.
View Absence Requests	View requests from students in their group. Cannot approve — approval is supervisor-only.
View Student List	Name, email, and group of students in their assigned group. No access to national ID or full personal data.
View Notifications	Receive exam date notifications for assigned courses.

TA CANNOT: Create or edit tracks/courses, publish grades, approve absence requests or KPIs, access other tracks or groups.



13. Training Manager Module

The Training Manager is the system super-admin. They inherit all permissions of all other roles and have no access restrictions within the portal.

13.1  Exclusive Capabilities
•Create and manage Branches, Intakes, Tracks.
•Assign Supervisors to tracks (one per track-intake).
•Create staff accounts (Supervisors, TAs, Student Affairs officers).
•Access any track, course, grade, or student record regardless of track assignment.
•Override or edit any grade, attendance record, or KPI decision.
•Publish grades and exam dates on behalf of any supervisor.
•Generate system-wide reports: grade distributions, absence rates, KPI completion.
•Configure system settings: academic calendar, absence thresholds, KPI requirements.
•Broadcast notifications to any role, track, or group.
•Deactivate or reactivate any user account.

16. Angular Module Map

Angular 17+ uses standalone components with lazy-loaded routing. Each role has its own feature module/folder. The router loads components on demand based on the authenticated user's role claim.

16.1  Route Structure
Route	Module / Component
/login	AuthModule → LoginComponent
/dashboard	Shared → RoleDashboardComponent (redirects by role)
/admin/**	TrainingManagerModule (canActivate: TrainingManagerGuard)
/supervisor/**	SupervisorModule (canActivate: SupervisorGuard)
/ta/**	TAModule (canActivate: TAGuard)
/affairs/**	StudentAffairsModule (canActivate: StudentAffairsGuard)
/student/**	StudentModule (canActivate: StudentGuard)

16.2  Key Angular Services
•AuthService — login, logout, token refresh, role extraction from JWT.
•TrackService — CRUD for branches, intakes, tracks, groups.
•CourseService — CRUD for courses; hours configuration.
•GradeService — read/write grades; publish action.
•AttendanceService — record sessions; fetch absence summaries.
•AbsenceRequestService — submit and review requests.
•ExamService — schedule exams; publish; corrective exam management.
•KpiService — submit certificate/freelance KPIs; review.
•NotificationService — fetch inbox; mark as read; real-time via SignalR (optional).
•UserService — create/edit/deactivate user accounts.

16.3  Angular Technical Stack
•Angular 17+ — standalone components (no NgModule).
•Signals — reactive state management.
•loadComponent() — lazy-loaded routes.
•inject() — dependency injection in functions and guards.
•HTTP Interceptors — auto-attach JWT bearer token.
•Route Guards — canActivate per role using role claim from token.
•Reactive Forms — FormBuilder with validators.
•Angular Material — UI component library.
•ngx-file-upload or native File API — for KPI proof uploads.
